# Test Case Sheet — DAG/Code Deployment Failure Scenarios

| # | Test Case Name | Details | Simple Explanation | Expected RCA | Trigger Command | OR Link |
|---|----------------|---------|-------------------|--------------|-----------------|---------|
| TC-1 | Bad DAG Deploy — Syntax Error | DAG file `pricing_margin_calculator_dag.py` has an incomplete dict literal (simulates truncated upload). Composer marks it as "Broken DAG" at parse time. Error: `SyntaxError: invalid syntax` | Someone pushed a new DAG file but it got corrupted or had merge conflicts. The DAG Processor can't parse it. | Bad code deployment: file was truncated or contains unresolved merge conflict. The syntax error at line N prevents Composer from loading the DAG. Redeploy the correct version of the file. | `gsutil cp pricing_margin_calculator_dag.py $DAG_PATH/` | |
| TC-2 | Missing Module — Undeployed Dependency | DAG file `store_replenishment_optimizer_dag.py` imports `mfm_inventory_utils` which doesn't exist in the environment. Composer marks it as "Broken DAG" at parse time. Error: `No module named 'mfm_inventory_utils'` | Someone deployed a new DAG but forgot to also deploy the shared Python package it depends on. | Missing dependency: DAG requires `mfm_inventory_utils` module which was not installed in the Composer environment. The dependency package needs to be deployed alongside the DAG. | `gsutil cp store_replenishment_optimizer_dag.py $DAG_PATH/` | |
| TC-3 | Breaking Change in Shared Library | DAG file `inventory_allocation_engine_dag.py` parses fine, but at runtime calls `calculate_allocation_priority(safety_stock_factor=...)` which was renamed in a library update. Error: `TypeError: got an unexpected keyword argument 'safety_stock_factor'` | The shared MFM library was updated and a function signature changed, but the DAGs calling it weren't updated at the same time. | Breaking change in shared library: `mfm_allocation_core.calculate_allocation_priority()` had parameter `safety_stock_factor` renamed to `buffer_multiplier` in a recent update. The DAG needs to be updated to match the new function signature. | `gsutil cp inventory_allocation_engine_dag.py $DAG_PATH/` then wait 2 min, then `gcloud composer environments run inventory-pipeline --location us-central1 dags trigger -- inventory_allocation_engine` | |
| TC-4 | Version Incompatibility — Deprecated Airflow API | DAG file `fulfillment_routing_pipeline_dag.py` parses fine, but at runtime tries to import from `airflow.contrib.operators.bigquery_operator` which was removed in Airflow 2.6+. Error: `ModuleNotFoundError: No module named 'airflow.contrib.operators.bigquery_operator'` | Composer environment was upgraded to Airflow 2.6+ but this DAG still uses the old `airflow.contrib` import path that was removed. | Version incompatibility after Composer upgrade: DAG uses `airflow.contrib.operators.bigquery_operator` which was removed in Airflow 2.x. The import should be updated to `airflow.providers.google.cloud.operators.bigquery`. | `gsutil cp fulfillment_routing_pipeline_dag.py $DAG_PATH/` then wait 2 min, then `gcloud composer environments run inventory-pipeline --location us-central1 dags trigger -- fulfillment_routing_pipeline` | |

## Notes

- **$DAG_PATH**: Get with `gcloud composer environments describe inventory-pipeline --location=us-central1 --format="value(config.dagGcsPrefix)"`
- **Cleanup**: After each scenario, run `gsutil rm $DAG_PATH/<dag_file>` and close the alert incident in GCP Console
- **Wait 5 min** between scenarios (alert rate limit)
- **OR Link**: Fill in OpsRabbit investigation link after each run
